Privacy Policy

This policy explains how MenswearStyle.store and the internal Google Ads campaign workflow tool handle Google OAuth authorization, Google Ads API data, retention, deletion and user control.

Policy Summary

Developer
Lian Shuai
Contact
lianwwe@gmail.com
Tool Scope
Internal Google Ads Search Campaign workflow

1. Who Maintains This Website

MenswearStyle.store is maintained by Lian Shuai, an independent developer. The website contains menswear editorial content and a dedicated developer project area for an internal Google Ads campaign workflow tool.

2. Google OAuth Usage

The internal tool uses Google OAuth 2.0 so an authorized user can grant access to approved Google Ads accounts. The tool does not request, receive, or store Google account passwords.

OAuth authorization is used only to support the Google Ads API functions described on this website: Search Campaign creation, campaign management and reporting for authorized accounts.

3. Google Ads API Data Collected Or Accessed

Data Type Purpose Use Limitation
Google Ads account identifiers Confirm the authorized account context before campaign workflow actions. Used only for authorized account selection and workflow execution.
Search Campaign configuration Prepare, validate, create and manage approved Search Campaign records. Used only for internal campaign workflow operations.
Keyword, ad group and status data Support campaign preparation, quality validation and campaign management. Not sold, rented, or used for unrelated profiling.
Reporting metrics Generate internal performance reports for authorized Google Ads accounts. Used only for reporting, review and campaign management decisions.

4. How Data Is Used

Google Ads API data is used to connect campaign preparation, validation, authorized review, deployment and reporting workflows. Data is not used for unrelated advertising, user profiling, resale, lead generation, or third-party marketing.

5. Account Authorization

The tool is intended for internal users only. Access is limited to Google Ads accounts that have been explicitly authorized through OAuth 2.0 by a permitted user. The tool does not support unauthorized account access.

6. Data Retention

Campaign workflow records may be retained only as needed for internal validation, campaign management, audit review and reporting. Records that are no longer required for these purposes can be deleted upon request.

7. Data Sharing

Google Ads account data, campaign data and reporting data are not sold, rented, or shared for unrelated advertising, profiling, or third-party marketing. Data is used only for the internal workflow described on this website.

8. Limited Use Statement

Use of Google user data is limited to the practices disclosed in this policy and to the internal Google Ads campaign workflow functionality requested by authorized users.

9. Security Practices

  • Google passwords are not requested or stored.
  • OAuth 2.0 is used for Google account authorization.
  • API usage is limited to authorized Google Ads accounts.
  • The tool does not use scraping or unauthorized access methods.
  • Campaign changes are reviewed before deployment.

10. Revocation And Deletion Requests

Users can revoke OAuth access from their Google account settings. To request deletion of stored workflow records associated with an authorized account, contact lianwwe@gmail.com.

11. Contact

For privacy, OAuth, Google Ads API, or data deletion questions, contact lianwwe@gmail.com.
